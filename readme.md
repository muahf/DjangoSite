# Библиотека на Django
Простой проект на Django, демонстрирующий основные возможности фреймворка: создание, чтение, обновление и удаление записей (CRUD), работа с моделями, формами, шаблонами и фильтрация данных, а также интеграция 3D-моделей с использованием Three.js.

## Возможности

### Основной функционал библиотеки
* Вывод информации - главная страница со статистикой и последними добавленными книгами
* Добавление данных - формы для создания книг и авторов
* Просмотр деталей - подробная информация о каждой книге
* Редактирование и удаление - полный CRUD функционал
* Поиск и фильтрация - расширенный поиск по нескольким критериям

### 3D визуализация кроссовок
* **Интерактивная 3D модель** - просмотр модели Nike Air TN в браузере с возможностью вращения, масштабирования и панорамирования
* **Автоматическое вращение** - модель плавно вращается для демонстрации со всех сторон
* **Динамическая загрузка** - индикатор прогресса загрузки модели
* **Качественное освещение** - многоуровневая система освещения с тенями для реалистичного отображения
* **Адаптивный рендеринг** - автоматическая подстройка под размер окна браузера
* **Поддержка формата glTF** - оптимизированный формат для веб-визуализации с текстурами высокого качества

## Технологии
* **Backend**: Django 6.0+
* **Frontend**: HTML5, CSS3, JavaScript
* **3D графика**: Three.js (GLTFLoader, OrbitControls)
* **База данных**: SQLite (по умолчанию)
* **Форматы 3D моделей**: glTF/GLB с поддержкой текстур

## Установка и запуск

1. Клонировать репозиторий
    ```bash
    git clone https://github.com/Julia-Mamsheva/ProjectDjango
    cd projectdjango
    ```

2. Создать виртуальное окружение
    ```bash
    # Windows
    python -m venv venv
    venv\Scripts\activate
    
    # Linux/Mac
    python3 -m venv venv
    source venv/bin/activate
    ```

3. Установить зависимости
    ```bash
    pip install -r requirements.txt
    ```

4. Применить миграции
    ```bash
    python manage.py makemigrations
    python manage.py migrate
    ```

5. Создать суперпользователя (для доступа к админке)
    ```bash
    python manage.py createsuperuser
    ```

6. Разместить 3D модель
    ```
    Скопируйте файлы модели в папку:
    /static/sneaker_viewer/models/
    
    Структура папок:
    static/
    └── sneaker_viewer/
        ├── js/
        │   └── main.js        # Скрипт для Three.js
        ├── css/
        │   └── style.css       # Стили (опционально)
        └── models/
            └── scene.gltf       # 3D модель
            └── scene.bin         # Бинарные данные модели
            └── textures/         # Папка с текстурами
                └── *.png
    ```

7. Запустить сервер
    ```bash
    python manage.py runserver
    ```

## Команды для создания superuser

### 1. Создание суперпользователя (администратора)

```bash
python manage.py createsuperuser
```

После ввода этой команды вас попросят ввести:
- **Username** (имя пользователя) - например: `admin`
- **Email address** (email) - например: `admin@example.com`
- **Password** (пароль) - введите пароль (при вводе символы не отображаются)
- **Password (again)** - повторите пароль

Пример:
```bash
(project) user@MacBook-Pro projectDjango % python manage.py createsuperuser
Имя пользователя: admin
Адрес электронной почты: admin@example.com
Password: 
Password (again): 
Superuser created successfully.
```

### 2. Запуск сервера разработки

```bash
python manage.py runserver
```
### 3. Переход на сайт и в админку

После запуска сервера откройте браузер и перейдите по адресам:

| **Админ-панель** | http://127.0.0.1:8000/admin/ |


### 4. Вход в админ-панель

1. Перейдите по адресу http://127.0.0.1:8000/admin/
2. Введите логин и пароль, которые вы создали на шаге 1
3. Вы попадете в административную панель Django, где сможете управлять моделями




## Функционал по страницам

### Основные страницы библиотеки
* **Главная страница (`/`)**
    * Статистика: общее количество книг, доступных книг, авторов
    * Последние добавленные книги
    * Навигация по разделам

* **Список книг (`/books/`)**
    * Каталог всех книг с краткой информацией
    * Поиск по названию, автору и ISBN
    * Фильтрация по автору, году издания
    * Фильтр "только доступные"
    * Кнопки для просмотра, редактирования и удаления

* **Детальная страница книги (`/books/<id>/`)**
    * Полная информация о книге
    * Ссылки на редактирование и удаление

* **Добавление/редактирование книги (`/books/create/` и `/books/<id>/update/`)**
    * Форма со всеми полями книги
    * Валидация данных
    * Сообщения об успешном сохранении

* **Удаление книги (`/books/<id>/delete/`)**
    * Подтверждение удаления
    * Защита от случайного удаления

* **Список авторов (`/authors/`)**
    * Все авторы с информацией
    * Количество книг каждого автора
    * Ссылки на книги автора

* **Добавление автора (`/authors/create/`)**
    * Форма для добавления нового автора

### Страница 3D визуализации
* **3D модель кроссовка (`/sneaker/`)**
    * Интерактивная модель Nike Air TN с текстурами
    * Управление мышью:
        - Левая кнопка + движение: вращение модели
        - Правая кнопка + движение: панорамирование
        - Колесико мыши: масштабирование
    * Автоматическое вращение для полного обзора
    * Индикатор загрузки с отображением процентов
    * Реалистичное освещение и тени
    * Адаптация под любой размер экрана

## Настройка 3D модели

Если вы хотите использовать свою 3D модель:

1. Экспортируйте модель в формате glTF (рекомендуется) или GLB
2. Убедитесь, что все текстуры находятся в правильных папках
3. Отредактируйте путь к модели в файле `templates/books/test.html`:
   ```html
   <script>
       window.modelPath = "{% static 'sneaker_viewer/models/your_models.gltf' %}";
   </script>
   ```
4. При необходимости отрегулируйте масштаб в `main.js`:
   ```javascript
   model.scale.set(0.1, 0.1, 0.1); // Измените значение для вашей модели
   ```

## Требования к 3D модели
* Формат: glTF (предпочтительно) или GLB
* Максимальный размер: рекомендуется до 50 МБ для оптимальной загрузки
* Текстуры: поддерживаются форматы PNG, JPEG
* Полигонаж: оптимизирован для веба (рекомендуется до 100k полигонов)

1. Создайте папку

    mkdir test_project
    ```

    ```

6. Создание проекта
    ```bash
    django-admin startproject first_project

7. Перейдите в папку для запуска проекта

    ```

8. Создание приложениея внутри проекта
    ```bash
    python manage.py startapp module_project
    ```

9. Команда для запуска вашего проекта
    ```bash
    python manage.py runserver
    ```

## Устранение неполадок с 3D моделью

### Модель не отображается
1. Откройте консоль браузера (F12) и проверьте ошибки
2. Убедитесь, что файлы модели находятся по правильному пути
3. Проверьте, что все текстуры загружаются (статус 200 в сети)
4. Попробуйте изменить масштаб модели в main.js

### Ошибки CORS
Если модель загружается с ошибками CORS, убедитесь что все файлы обслуживаются через Django статические файлы.

### Медленная загрузка
* Оптимизируйте модель, уменьшите количество полигонов
* Сожмите текстуры
* Используйте формат GLB вместо GLTF с отдельными файлами
